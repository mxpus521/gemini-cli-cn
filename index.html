简介

Gemini-CLI 是 Google 推出的革命性命令行 AI 助手，它不仅仅是一个聊天工具，更是一个深度集成到您开发环境中的智能编程伙伴。

💡 核心理念

Gemini-CLI 的设计理念是"终端优先"，让 AI 成为您工作流程的自然延伸，而不是另一个需要管理的工具。

主要特性

  • 终端原生体验：无缝集成到您的命令行环境
  • 智能上下文管理：自动理解项目结构和编码规范
  • 强大的工具集成：文件操作、命令执行、网络搜索等
  • 高度可定制：支持自定义命令和扩展
  • 隐私优先：您的代码和数据安全始终是首要考虑

安装与配置

系统要求

  • Node.js v20.0.0 或更高版本
  • npm 或 yarn 包管理器
  • 支持的操作系统：macOS、Linux、Windows

安装步骤

1. 检查 Node.js 版本

bash 
node --version
npm --version

2. 全局安装 Gemini-CLI

bash 
npm install -g @google/gemini-cli
⚠️ 权限问题

如果遇到权限错误，请不要使用 sudo。建议配置 npm 的全局包路径，或使用 nvm 管理 Node.js 版本。

3. 验证安装

bash 
gemini --version

首次运行

启动 Gemini-CLI

bash 
gemini

身份验证

首次运行时，系统会提示您进行身份验证：

  1. 选择 "Login with Google" 选项
  2. 浏览器将自动打开 Google 登录页面
  3. 完成授权后返回终端
  4. 看到欢迎消息表示配置成功
✅ 恭喜！

您已成功配置 Gemini-CLI，现在可以开始使用了！

核心交互模式

1. 自然语言对话

最基础的交互方式，直接输入您的问题或指令：

gemini 
请帮我用 Python 写一个快速排序算法

2. @ 命令：文件上下文注入

使用 @ 符号将文件或目录内容作为上下文：

gemini 
请根据 @./src/utils.js 的代码风格，为新函数添加注释

3. ! 命令：执行 Shell 命令

直接在 CLI 内执行终端命令：

gemini 
!git status
!npm test

斜杠(/)命令大全

会话与历史管理

命令 说明 示例
/chat save 保存当前会话 /chat save my-session

高级工作流程

1. 代码调试工作流

workflow 
# 1. 载入问题代码
请分析 @./src/api.py 中的错误

# 2. 运行测试
!pytest tests/test_api.py

# 3. 获取修复建议
根据测试结果，请提供修复方案

# 4. 应用修复
/copy  # 复制生成的代码

2. 文档生成工作流

workflow 
# 1. 初始化项目配置
/init

# 2. 设置文档规范
/memory add "所有文档使用 Markdown 格式，包含代码示例"

# 3. 生成文档
请为 @./src/ 目录生成 API 文档

# 4. 保存会话
/chat save docs-generation

3. 代码审查工作流

workflow 
# 1. 查看最近的更改
!git diff HEAD~1

# 2. 请求代码审查
请审查上述代码更改，关注：
- 潜在的 bug
- 性能问题
- 最佳实践

# 3. 生成审查报告
请生成一份详细的代码审查报告

深度自定义

配置 settings.json

运行 /settings 打开配置文件：

json 
{
  "model": "gemini-1.5-pro-latest",
  "temperature": 0.7,
  "maxTokens": 8192,
  "tools": {
    "shell": {
      "execute": true,
      "requireConfirmation": true
    },
    "fileSystem": {
      "read": true,
      "write": true
    }
  },
  "theme": "dracula",
  "editor": "vim"
}

创建自定义命令

在 ~/.gemini/commands/ 目录创建可执行脚本：

bash 
#!/bin/bash
# 文件: ~/.gemini/commands/deploy

echo "🚀 开始部署..."
git push origin main
npm run build
npm run deploy
echo "✅ 部署完成！"

赋予执行权限：

bash 
chmod +x ~/.gemini/commands/deploy

现在可以使用 /deploy 命令了！

创建项目模板

创建 GEMINI.md 文件定义项目规范：

markdown 
# 项目规范

## 技术栈
- Frontend: React + TypeScript
- Backend: Node.js + Express
- Database: PostgreSQL

## 编码规范
- 使用函数式编程风格
- 所有函数必须有 JSDoc 注释
- 使用 ESLint 和 Prettier
- 测试覆盖率必须 > 80%

## 文件结构
src/
components/ # React 组件
services/ # API 服务
utils/ # 工具函数
types/ # TypeScript 类型定义

## Git 规范
- 使用 conventional commits
- 分支命名: feature/*, bugfix/*, hotfix/*
- PR 必须通过所有测试

常见问题解答

Q: 如何更新到最新版本？

运行 npm update -g @google/gemini-cli 即可更新到最新版本。

Q: 支持哪些编程语言？

Gemini-CLI 支持所有主流编程语言，包括但不限于 JavaScript、Python、Java、Go、Rust、C++等。

Q: 如何处理大文件？

对于大文件，建议使用 @ 命令只引入相关部分，或使用 /compress 压缩上下文。

Q: 数据安全性如何？

所有数据传输都使用 HTTPS 加密，Google 不会存储您的代码。详见 /privacy 命令。

Q: 如何贡献代码？

访问 GitHub 仓库，阅读贡献指南并提交 PR。

故障排除

常见错误及解决方案

1. 权限错误

错误: EACCES: permission denied

解决方案：配置 npm 全局包路径或使用 nvm 管理 Node.js

2. 网络连接问题

错误: Network error

解决方案：检查代理设置，确保能访问 Google 服务

3. Token 限制

警告: Context too large

解决方案：使用 /compress 压缩历史或 /chat clear 清空会话
